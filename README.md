# יומן פרויקטים — לבנט

אפליקציית דפדפן לניהול טיים ליין רישוי של פרויקטי נדל"ן, עם **סנכרון בין כל המכשירים**.

## שימוש

נכנסים לכתובת הקבועה — https://drimi42312.github.io/project-journal/

בכניסה הראשונה בכל מכשיר מזינים את **סיסמת הצוות** פעם אחת. מרגע זה כל פרויקט וכל עריכה מסתנכרנים אוטומטית בין כל המכשירים והדפדפנים של הצוות — מי שמעדכן במכשיר אחד, העדכון מופיע מיד אצל כולם.

## יכולות

- ריבוי פרויקטים — כל פרויקט חדש מקבל טיים ליין רישוי מלא של 15 שלבי ברירת מחדל (מתכנון ראשוני ועד אגרות והשבחה)
- תאריך התחלה וסיום לכל שלב, עם חישוב משך אוטומטי
- הוספת עיכוב בין כל שני שלבים — שדה חופשי לגורם המעכב (יזם, עירייה, יועץ...) וספירת משך העיכוב; עיכוב ללא תאריך סיום נספר עד היום
- הוספת שלבים מותאמים אישית בכל נקודה בציר, מחיקת שלבים ועריכת שמות
- החלפת קטגוריית שלב (תכנון / עירייה / אבן דרך) בלחיצה על העיגול
- סיכום: עבודה נטו, סך עיכובים, ומשך מקצה לקצה
- ייצוא כל הפרויקטים לקובץ אקסל (גיליון לכל פרויקט, כולל עיכובים וסיכומים)
- גיבוי ושחזור JSON

## איך הסנכרון עובד

- **Frontend** — קובץ `index.html` יחיד, מתארח ב-GitHub Pages (סטטי, ללא בנייה).
- **Backend** — שרת סנכרון קטן (`server/`) מתארח ב-Railway, עם נפח אחסון קבוע (volume) כך שהנתונים לא נמחקים.
- **סנכרון חי** — Server-Sent Events: כל שינוי משודר מיידית לכל המכשירים הפתוחים.
- **עבודה לא-מקוונת** — עריכות נשמרות מקומית (localStorage) וגם אם אין רשת, ומסתנכרנות אוטומטית כשהחיבור חוזר.
- **אבטחה** — כל הבקשות דורשות את סיסמת הצוות (bearer token). הסיסמה נשמרת רק במכשיר, לא בקוד.
